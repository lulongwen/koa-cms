## 数据库类型

1. 关系型数据库优点：关联查询，排序方便
    - 缺点：字段修改不灵活

2. key-value存储数据库优点：详情字段灵活
    - 缺点：排序不灵活，关联查询

3. 垂直切割：把不同的功能部署到不同的服务器
4. 水平切割：相同的功能部署多台服务器

5. 系统设计
    - 信息采集 - 内容编辑 - 格式化
    

### 数据库范式

1. 字段的冗余性，不存在数据冗余和传递依赖
2. 记录的唯一性，有唯一标识：主键；其他属性都依赖主键
3. 属性的原子性，每一列都不可再拆分
